# dddlib

`dddlib` は、Domain-Driven Design を実装するための基盤をまとめた Python ライブラリです。

このライブラリは、アプリケーション固有のビジネスロジックそのものを提供するのではなく、DDD の設計で繰り返し登場する共通概念を、型安全に扱いやすい形で提供することを目的としています。

## 位置づけ

`dddlib` は、ドメイン層で必要になる以下のような要素を共通化します。

- 値オブジェクト
- エンティティ
- 集約ルート
- ドメインイベント
- 識別子
- ドメインエラー
- メッセージ

これらを毎回アプリケーションごとに実装すると、同じような制約や検証、等価比較、再構築処理が散在しやすくなります。
`dddlib` は、その繰り返しを減らし、ドメインモデルそのものの設計に集中しやすくするための土台です。

## 設計方針

`dddlib` は次の方針で設計されています。

- `dataclass` を前提に、値の保持と不変性を扱いやすくする
- `mypy` による型チェックを通しやすい実装にする
- `ruff` の厳しめのルールに合わせて、明示的で読みやすいコードを保つ
- 具体的な業務処理は持たず、ドメインの共通基盤に限定する
- `dddlib_cli` と分離し、ライブラリ利用と CLI 利用を切り分ける

このため、`dddlib` はフレームワークではなく、ドメインモデルのための薄い基盤層として使うことを想定しています。

## 提供する機能

### 値オブジェクト

値オブジェクトは、ID ではなく値そのものを基準に同一性を扱うオブジェクトです。

`dddlib` では、値オブジェクトを作る際に必要になりやすい以下の処理をまとめています。

- 生成時の検証
- 辞書化
- 表示用文字列への変換
- 不変な値としての保持

`FrozenBase` を使うことで、`dataclass(frozen=True)` 相当の振る舞いを自然に扱えます。

### エンティティ

エンティティは、識別子によって同一性を持つオブジェクトです。

`dddlib` のエンティティ基盤は、次のような処理をサポートします。

- 識別子を持つオブジェクトの生成
- 再構築用の内部生成
- 同一性に基づく比較
- 識別子の取得

値オブジェクトと異なり、エンティティはライフサイクルを持つため、同一性の判定を明確に分離できることが重要です。

### 集約ルート

集約ルートは、ドメインの整合性を守るための入口です。

`dddlib` の集約ルートは、エンティティの特性を引き継ぎながら、ドメインイベントを蓄積・取得できるようにしています。

- イベントの追加
- イベントの取得と取り出し
- 複数イベントのまとめ追加

これにより、ドメイン操作の結果として発生したイベントを安全に扱いやすくなります。

### ドメインイベント

ドメインイベントは、ドメイン内で起きた重要な出来事を表します。

`dddlib` では、イベントを単なる辞書や文字列ではなく、型を持つオブジェクトとして扱うことを重視しています。
イベントの内容を明示的なクラスとして表すことで、イベント駆動の処理や監査、通知処理に接続しやすくなります。

### 識別子

識別子は、エンティティや集約を一意に扱うための値です。

`dddlib` には、UUID ベースの識別子を中心に、値の検証や変換をまとめるための基底が用意されています。

これにより、`str` や `int` をそのまま散らすのではなく、意味を持った型として ID を扱いやすくなります。

### エラーとメッセージ

`dddlib` では、ドメイン層で扱うエラーとメッセージも共通化しています。

- `DDDError` は、コードとメッセージを持つドメインエラーの基盤です
- `Message` は、メッセージ本文の生成と整形を扱います
- `DDDMessage` は、コード付きの定義済みメッセージを表します

この仕組みを使うと、エラーを単なる例外メッセージに閉じ込めず、アプリケーション全体で一貫した扱いにしやすくなります。

## 代表的な利用イメージ

`dddlib` は、たとえば次のような構成で使うことを想定しています。

- ドメインの ID を `UUIDIdentifier` から派生して定義する
- 値オブジェクトに検証ロジックを持たせる
- エンティティで同一性を ID ベースに判定する
- 集約ルートからドメインイベントを発行する
- ドメインエラーで業務上の失敗を明示する

この設計により、アプリケーション層やインフラ層に寄せすぎず、ドメインの言葉でモデルを組み立てやすくなります。

## CLI との関係

`dddlib` の本体はライブラリです。

CLI を使ってプロジェクト作業を支援したい場合は、別パッケージの [dddlib_cli](../dddlib_cli/README.md) を参照してください。

## 導入の考え方

導入時は、次の順で考えると整理しやすくなります。

1. まず、ドメインの識別子をどう表現するか決める
1. 次に、値オブジェクトとして閉じるべき概念を切り出す
1. その上で、エンティティと集約ルートを定義する
1. 最後に、イベントとエラーの表現をそろえる

`dddlib` は、この流れを破壊するのではなく、実装を揃えるための道具として使うのが適切です。

## 注意点

- `dddlib` は完成した業務アプリケーションではなく、共通基盤です
- そのまま使うだけではドメインは成立しないため、各プロジェクトで具体的なモデル定義が必要です
- CLI の詳細は [dddlib_cli README](../dddlib_cli/README.md) に分離しています
